COMANDO: /comandos

O QUE É

Índice de todos os comandos disponíveis, com uma linha sobre cada um.
É o menu de consulta rápida do sistema.

QUANDO USAR

Use quando não lembrar qual comando aciona o quê, ou ao mostrar o sistema a outra IA pela primeira vez.

POR QUE USAR

Centraliza a referência e economiza esforço de memória.

---

LISTA ATIVA

/mpe → aplica silenciosamente o protocolo SRC CMP TEC CAL SYN VAL STR na próxima resposta. Usar para temas jurídicos, acadêmicos, técnicos, regulatórios, comparativos, financeiros, estatísticos, científicos, auditorias e análises com risco factual.

/mpe+ → /mpe com pesquisa multilíngue ampliada. Sozinho: além de PT-BR e inglês, adiciona línguas úteis ao tema. Com códigos, força línguas específicas. Ex.: /mpe+ de fr zh.

/consolidar → gera resumo de continuidade da conversa em bloco único, para retomar noutra conversa/IA sem perder contexto. Ao final, pode sugerir gerar o prompt de inicialização da próxima conversa.

/nova-conversa → gera o prompt de inicialização da próxima conversa, no formato IDENTIDADE E ESTILO / ARQUITETURA / SECRETS / DECISÕES / DIFICULDADES E SOLUÇÕES / PENDÊNCIAS / REGRAS PARA A PRÓXIMA IA / SEGURANÇA. Normalmente vem depois de /consolidar.

/id → liga o carimbo de identificação e proveniência no topo de toda resposta. /id off desliga.

/friendly → comando adaptativo de acessibilidade cognitiva. Ajusta a forma da resposta sem mexer no rigor técnico. TDAH como base fixa, explicitude ampliada em tarefas complexas, compressão leve em tarefas simples.

/f → alias curto de /friendly.

/f + → explicitude ampliada (premissas, riscos, distinções, literalidade).

/f - → resposta mínima (só o essencial, sem aprofundar).

/f b → burnout puro (máx. dois parágrafos curtos, sem opções empilhadas, tom calmo, sem próximo passo obrigatório).

/rodape → ativa rodapé estendido com idiomas pesquisados, limitações relevantes, premissas não confirmadas e mentoria linguística detalhada.
/rodape off desativa. Alias: /r e /r off.

/comandos → este arquivo.

REGRAS DE USO

1. Comandos começam com barra.
2. Ao receber comando, ler a definição completa do arquivo correspondente antes de executar, quando o arquivo estiver acessível.
3. Não inventar comandos fora desta lista.
4. Se o comando mencionado não constar na lista, avisar antes de tentar adivinhar.
5. Se o arquivo de definição não estiver acessível: "não consigo validar sem o arquivo efetivamente anexado nesta conversa".
