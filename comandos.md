COMANDO: /comandos

O QUE É: índice dos comandos ativos.

LISTA ATIVA

`/mpe` — aplica rigor técnico.
`/mpe+` — pesquisa multilíngue ampliada.
`/consolidar` — resumo de continuidade.
`/nconversa` — prompt para nova conversa.
`/id` — cabeçalho de identificação.
`/friendly` ou `/f` — acessibilidade cognitiva.
`/rodape` ou `/r` — rodapé estendido.
`/comandos` — este índice.
`/pafe` — fluxo de estudo conforme `pafe/README.md`.
`/pafe html` — exatamente um HTML offline, sem áudio, sem impressão e sem arquivos auxiliares.
`/pafe audio` — MP3 real por assunto; ler primeiro `pafe/audio_modos.md`.
`/pafe html audio` e `/pafe audio html` — um HTML independente, sem áudio, mais N MP3 independentes, um por assunto.
`/pafe audio local` ou `/pafe audio script` — força script local mínimo.
`/pafe audio pacote` — força pacote técnico completo.

REGRAS DE USO

1. Ler a definição completa do arquivo correspondente quando acessível.
2. Não inventar comando fora da lista.
3. A linguagem natural do usuário prevalece sobre a necessidade de token exato.
4. Expressões como “conforme informação anterior”, “esses 28 arquivos”, “um por assunto” e “para executar no Linux” recuperam decisões anteriores.
5. Para áudio, `pafe/audio_modos.md` decide a rota e prevalece sobre `pafe/audio.md`.
6. Criar arquivo para download dentro da conversa não exige autorização adicional quando o usuário já pediu o artefato.
7. Exigem autorização expressa: escrita em GitHub/Drive/Box, API paga, envio sensível a terceiro, exclusão ou substituição externa.
8. Nunca usar voz robótica como fallback.
9. Nunca gerar `master_audio.mp3` por padrão.
10. Se a geração direta falhar e o Linux local conhecido for compatível, gerar automaticamente o script local mínimo.
